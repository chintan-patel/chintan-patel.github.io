How to update the stale git fork

04 May 2016 on git, git fork

To contribute to the git repository, you must be forking that repository under your account.

For first time when you contribute and create the Pull Request the branches of parent and your repository are in sync. But what happens when the parent repository is updated and your forked repository is out-of-date?

There are two ways to update the fork:

  • Using Github
  • Using git cli

Using Github,

  1. Create a Pull Request
  2. Under Comparing changes view, do following changes:
    • Change "base fork" to your repository with develop or whatever branch required.
    • Change "head fork" to parent repository with develop or whatever branch required.
  3. Create Pull Request
  4. Merge the changes after proper verification and approval.

Using git cli

  1. Go to the repository and checkout the develop branch.
  2. Update the branch - git pull origin develop
  3. git pull https://github.com/otheruser/repo.git develop
  4. git push origin develop
